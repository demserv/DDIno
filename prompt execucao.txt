5. REGRAS DE CODIFICAÇÃO OBRIGATÓRIAS
=====================================================================

1. Linguagem:
   - C para firmware
   - Headers compatíveis com C++
   - Usar guardas:

      #ifdef __cplusplus
      extern "C" {
      #endif

      ...

      #ifdef __cplusplus
      }
      #endif

2. Retornos:
   - Funções críticas retornam esp_err_t
   - Nunca ignorar esp_err_t

3. Logs:
   - Usar ESP_LOGI/W/E com TAG
   - Logar falhas críticas

4. Segurança:
   - Não usar strcpy, sprintf, gets
   - Usar snprintf, strncpy com cuidado, validação de limite

5. Concorrência:
   - Recursos compartilhados com mutex/semaphore
   - Timeouts claros
   - Sem busy-wait
   - Sem vTaskDelay(0)

6. ISR:
   - Sem alocação dinâmica
   - Sem bloqueio

7. Memória:
   - Evitar malloc em loop crítico
   - Monitorar stack/heap onde crítico

8. Organização:
   - Separar HAL, drivers, services, FSMs, UI e API
   - UI/API não acessam hardware diretamente
   - Drivers não conhecem lógica de negócio

9. Build:
   - CMakeLists coerentes
   - Componentes próprios bem definidos
   - sdkconfig.defaults documentado
   - partitions.csv coerente com NVS/SD/OTA se aplicável
