# Governance — Monitor Aquário Inteligente

- **Code Review Policy:** Toda PR requer 1 approval de revisor designado.
- **Branch Strategy:** main → desenvolvimento, feature/* → features, fix/* → correções.
- **Release Process:** Tag semver (vX.Y.Z) + CHANGELOG.md.
- **Ownership Matrix:** Vide OWNERSHIP_AF4.md.
- **Change Request:** CR documentado via issue + PR link.
- **Test Gate:** PR só merge se idf.py build + idf.py test passarem.
