8. FASE 2 — CORRIGIR BUILD, STUBS E FUNÇÕES SEM RETORNO
================================================================================

Arquivos prioritários:
- firmware/drivers/driver_mcp3208.c
- firmware/drivers/driver_pzem.c
- firmware/drivers/driver_ds18b20.c
- firmware/drivers/driver_acs712.c
- firmware/drivers/driver_ad_keypad.c
- firmware/drivers/driver_mcp23017.c
- firmware/drivers/driver_relay.c
- CMakeLists.txt

Instruções:
1. Ative -Wall, -Wextra, -Werror e -Wreturn-type no CMake.
2. Rode build.
3. Para cada função não-void sem retorno, implemente lógica real ou retorne erro explícito documentado.
4. É proibido retornar ESP_OK em função não implementada.
5. Stub temporário só pode retornar ESP_ERR_NOT_SUPPORTED ou erro equivalente, com log ESP_LOGE e RTM como NON-COMPLIANT/PARTIAL.

Exemplo proibido:
```c
esp_err_t pzem_read_all(pzem_data_t *data)
{
    memset(data, 0, sizeof(*data));
    return ESP_OK; // PROIBIDO
}
```

Exemplo temporário aceitável, apenas se inevitável:
```c
esp_err_t pzem_read_all(pzem_data_t *data)
{
    if (data == NULL) return ESP_ERR_INVALID_ARG;
    memset(data, 0, sizeof(*data));
    data->valid = false;
    ESP_LOGE(TAG, "pzem_read_all ainda nao implementado: retorno conservador");
    return ESP_ERR_NOT_SUPPORTED;
}
```
